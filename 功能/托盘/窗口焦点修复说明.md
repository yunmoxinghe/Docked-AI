# 窗口焦点修复说明

## 问题描述
有时从托盘图标或快捷键唤起窗口时，窗口虽然显示了，但没有获得焦点（无法直接输入）。

## 根本原因
Windows 系统对 `SetForegroundWindow` API 有严格的限制，防止应用程序随意抢占焦点。简单调用 `window.Activate()` 在某些情况下无法确保窗口获得焦点。

## 解决方案
创建了 `WindowHelper` 类，使用多种技术组合确保窗口能够成功获得焦点：

1. **线程输入附加**：通过 `AttachThreadInput` 将当前线程附加到前台窗口的线程，这是绕过 `SetForegroundWindow` 限制的关键技术
2. **Z-order 提升**：短暂设置窗口为 TOPMOST，然后取消，帮助窗口获得 Z-order 优先级
3. **多次调用**：在不同阶段多次调用 `SetForegroundWindow`，提高成功率

## 修改的文件
- `功能/托盘/WindowHelper.cs`（新建）：窗口焦点辅助类
- `功能/托盘/托盘图标管理器.cs`：使用 WindowHelper 设置焦点
- `功能/主窗口/显示隐藏/窗口宿主控制器.cs`：在所有显示窗口的地方添加焦点设置

## 技术细节
WindowHelper 使用的关键 Win32 API：
- `AttachThreadInput`：附加线程输入，允许跨线程设置焦点
- `SetForegroundWindow`：设置前台窗口
- `BringWindowToTop`：提升窗口 Z-order
- `SetWindowPos` with TOPMOST：临时提升窗口优先级
