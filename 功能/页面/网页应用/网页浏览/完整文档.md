# 网页浏览页面架构文档

## 概述

网页浏览页面采用模块化架构设计，将原本1400+行的单一文件重构为多个职责清晰的模块，提高代码的可维护性、可测试性和可扩展性。

## 架构设计

### 目录结构

```
网页浏览/
├── Constants/              # 常量配置
│   └── WebBrowserConstants.cs
├── Services/               # 服务层
│   ├── ColorService.cs
│   ├── ThemeColorExtractor.cs
│   └── WebViewTintScript.cs
├── Managers/               # 管理器层
│   ├── BarThemeManager.cs
│   ├── ResponsiveLayoutManager.cs
│   └── CornerRadiusManager.cs
├── 网页浏览页面.xaml       # UI定义
└── 网页浏览页面.xaml.cs    # 页面逻辑（协调层）
```

## 模块说明

### 1. Constants - 常量配置层

**WebBrowserConstants.cs**
- 集中管理所有魔法数字和配置常量
- 包括消息类型、浏览器参数、采样配置等
- 便于统一修改和维护

### 2. Services - 服务层

#### ColorService.cs
**职责**: 颜色处理核心服务

**主要功能**:
- `CalculateLuminance()` - 计算颜色亮度（WCAG标准）
- `GetContrastingForeground()` - 获取对比前景色
- `AdjustColorBrightness()` - 调整颜色亮度
- `CreateSecondaryColor()` - 创建次要颜色
- `AnimateColorChange()` - 颜色动画过渡
- `TryParseCssColor()` - 解析CSS颜色字符串

**特点**:
- 纯静态方法，无状态
- 易于单元测试
- 可在其他模块复用

#### ThemeColorExtractor.cs
**职责**: 从网页提取主题颜色

**主要功能**:
- `TryGetThemeColorAsync()` - 从meta标签提取主题色
- `TryScreenshotSamplingAsync()` - 截图采样获取颜色
- `SampleRegion()` - 采样指定区域的平均颜色

**特点**:
- 分层策略：优先meta标签，回退到截图采样
- 异步操作，不阻塞UI

#### WebViewTintScript.cs
**职责**: 管理WebView注入脚本

**主要功能**:
- `GetTintScript()` - 获取增强版取色脚本

**特点**:
- 脚本与代码分离
- 支持递归查找、渐变、图片背景等复杂场景
- 防抖优化，减少性能开销

### 3. Managers - 管理器层

#### BarThemeManager.cs
**职责**: 管理顶部栏和底部栏的主题

**主要功能**:
- `InitializeForegroundColors()` - 初始化前景色
- `ResetTintState()` - 重置取色状态
- `TryHandleTintMessage()` - 处理取色消息
- `ApplyBarTint()` - 应用栏着色
- `ApplySystemAccentColor()` - 应用系统强调色

**特点**:
- 封装所有颜色画刷
- 管理取色状态机
- 防闪烁逻辑

#### ResponsiveLayoutManager.cs
**职责**: 响应式布局管理

**主要功能**:
- `ApplyTopBarSpacing()` - 应用顶部栏间距
- `ApplyBottomBarLayout()` - 应用底部栏布局
- `CalculateButtonLayout()` - 计算按钮宽度和间距

**特点**:
- 纯静态方法
- 自适应算法
- 支持不同屏幕尺寸

#### CornerRadiusManager.cs
**职责**: 圆角同步管理

**主要功能**:
- `SyncCornerRadiusFromParent()` - 从父容器同步圆角
- `SyncDynamicCorners()` - 同步动态圆角到栏
- `FindParentCornerRadius()` - 查找父级圆角

**特点**:
- 递归查找父容器
- 自动回退到默认值
- 确保最小圆角

### 4. 页面层 - 网页浏览页面.xaml.cs

**职责**: 协调各个模块，处理UI事件

**主要职责**:
- 初始化各个管理器
- 处理WebView生命周期
- 响应用户交互
- 协调模块间通信

## 设计原则

### 1. 单一职责原则 (SRP)
每个类只负责一个明确的功能领域

### 2. 开闭原则 (OCP)
对扩展开放，对修改关闭。新增功能通过添加新类实现

### 3. 依赖倒置原则 (DIP)
高层模块不依赖低层模块，都依赖抽象

### 4. 关注点分离
UI、业务逻辑、数据处理分离

## 优势

### 1. 可维护性
- 代码结构清晰，易于理解
- 修改某个功能只需关注对应模块
- 减少代码重复

### 2. 可测试性
- 服务层和管理器层可独立测试
- 纯函数易于编写单元测试
- 模拟依赖更简单

### 3. 可扩展性
- 新增功能只需添加新的服务或管理器
- 不影响现有代码
- 易于集成新特性

### 4. 可读性
- 文件更小，更易阅读
- 命名清晰，职责明确
- 减少认知负担

## 使用示例

```csharp
// 解析CSS颜色
if (ColorService.TryParseCssColor("#FF5733", out var color))
{
    // 获取对比色
    var contrastColor = ColorService.GetContrastingForeground(color);
    
    // 调整亮度
    var brighterColor = ColorService.AdjustColorBrightness(color, 0.2);

```

```csharp
}
```

## 迁移指南

### 从旧代码迁移

1. **颜色处理**: 将所有颜色相关代码迁移到 `ColorService`
2. **主题提取**: 使用 `ThemeColorExtractor` 替代内联脚本
3. **布局管理**: 使用 `ResponsiveLayoutManager` 替代手动计算
4. **圆角同步**: 使用 `CornerRadiusManager` 统一管理

### 向后兼容

新架构完全向后兼容，可以逐步迁移：
1. 先引入新的服务类
2. 逐步替换旧代码
3. 保持功能不变

## 性能考虑

- 服务层使用静态方法，无实例化开销
- 管理器层按需创建，生命周期与页面一致
- 颜色动画使用 Storyboard，硬件加速
- 脚本注入一次性完成，后续无开销

## 完整示例：创建一个简单的网页浏览器

```csharp
using Docked_AI.Features.Pages.WebApp.Browser.Constants;
using Docked_AI.Features.Pages.WebApp.Browser.Managers;
using Docked_AI.Features.Pages.WebApp.Browser.Services;
using Microsoft.UI.Xaml;
using Microsoft.UI.Xaml.Controls;
using Microsoft.Web.WebView2.Core;
using System;
using System.Threading.Tasks;

namespace MyApp
{
    public sealed partial class SimpleBrowserPage : Page
    {
        private readonly BarThemeManager _themeManager = new();
        private Microsoft.UI.Xaml.Controls.WebView2? _webView;

        public SimpleBrowserPage()
        {
            InitializeComponent();
            InitializePage();
        }

        private void InitializePage()
        {
            // 初始化主题
            _themeManager.InitializeForegroundColors();
            ApplyThemeBrushes();
            
            // 监听事件
            SizeChanged += OnSizeChanged;
            Loaded += OnLoaded;
        }

        private void ApplyThemeBrushes()
        {
            TopBar.Background = _themeManager.TopBarBackgroundBrush;
            BottomBar.Background = _themeManager.BottomBarBackgroundBrush;
            TitleText.Foreground = _themeManager.TopBarForegroundBrush;
        }

        private async void OnLoaded(object sender, RoutedEventArgs e)
        {
            await InitializeWebViewAsync();
        }

        private async Task InitializeWebViewAsync()
        {
            if (_webView == null) return;

            // 创建 WebView2 环境
            var options = new CoreWebView2EnvironmentOptions
            {
                AdditionalBrowserArguments = string.Join(" ", 
                    WebBrowserConstants.BrowserArguments.OptimizedScrolling)
            };

            var environment = await CoreWebView2Environment.CreateWithOptionsAsync(
                null, null, options);
            await _webView.EnsureCoreWebView2Async(environment);

            // 设置
            _webView.CoreWebView2.Settings.IsWebMessageEnabled = true;
            
            // 事件
            _webView.CoreWebView2.NavigationStarting += OnNavigationStarting;
            _webView.CoreWebView2.NavigationCompleted += OnNavigationCompleted;
            _webView.CoreWebView2.WebMessageReceived += OnWebMessageReceived;
            
            // 安装取色脚本
            string script = WebViewTintScript.GetTintScript();
            await _webView.CoreWebView2.AddScriptToExecuteOnDocumentCreatedAsync(script);
        }

        private void OnNavigationStarting(object? sender, CoreWebView2NavigationStartingEventArgs e)
        {
            _themeManager.ResetTintState();
        }

        private async void OnNavigationCompleted(object? sender, CoreWebView2NavigationCompletedEventArgs e)
        {
            // 尝试提取主题色
            if (_webView?.CoreWebView2 != null)
            {
                var themeColor = await ThemeColorExtractor.TryGetThemeColorAsync(_webView.CoreWebView2);
                if (themeColor.HasValue)
                {
                    _themeManager.MarkThemeColorApplied();
                    _themeManager.ApplyBarTint(isTop: true, themeColor.Value);
                    _themeManager.ApplyBarTint(isTop: false, themeColor.Value);
                }
            }
        }

        private void OnWebMessageReceived(object? sender, CoreWebView2WebMessageReceivedEventArgs e)
        {
            string json = e.TryGetWebMessageAsString();
            _themeManager.TryHandleTintMessage(json);
        }

        private void OnSizeChanged(object sender, SizeChangedEventArgs e)
        {
            // 响应式布局
            ResponsiveLayoutManager.ApplyTopBarSpacing(TopBarGrid, TitlePanel, ActualWidth);
            
            // 圆角同步
            CornerRadiusManager.SyncDynamicCorners(this, TopBar, BottomBar);
        }
    }
}
```

## 性能优化

### 避免频繁调用

```csharp
// ❌ 不好：每次滚动都计算
private void OnScroll(object sender, ScrollEventArgs e)
{
    var color = SampleColor();
    _themeManager.ApplyBarTint(isTop: true, color);
}

// ✅ 好：使用防抖
private DispatcherTimer? _debounceTimer;

private void OnScroll(object sender, ScrollEventArgs e)
{
    _debounceTimer?.Stop();
    _debounceTimer = new DispatcherTimer { Interval = TimeSpan.FromMilliseconds(300) };
    _debounceTimer.Tick += (s, args) =>
    {
        _debounceTimer.Stop();
        var color = SampleColor();
        _themeManager.ApplyBarTint(isTop: true, color);
    };
    _debounceTimer.Start();
}
```

## 测试最佳实践

### 1. AAA 模式
```csharp
[Fact]
public void MethodName_Scenario_ExpectedResult()
{
    // Arrange - 准备测试数据
    var input = /* ... */;

    // Act - 执行被测试的方法
    var result = MethodUnderTest(input);

    // Assert - 验证结果
    Assert.Equal(expected, result);
}
```

### 2. 命名规范
- 方法名：`MethodName_Scenario_ExpectedResult`
- 清晰描述测试意图

- 不依赖其他测试
- 不依赖执行顺序
