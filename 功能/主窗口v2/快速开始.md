# 快速开始指南

欢迎使用主窗口 v2 架构！本指南将帮助您在 5 分钟内开始使用新的窗口管理系统。

## 🚀 立即开始

### 1. 创建窗口（已完成）

MainWindow 已经集成了新架构，您无需修改创建代码：

```csharp
// 在 App.xaml.cs 或 TrayIconManager 中
var window = MainWindowFactory.Create();
window.Activate();
```

### 2. 使用新的状态管理（需要更新）

#### 切换到全屏

**旧代码**:
```csharp
windowController.ToggleMaximize();
```

**新代码**:
```csharp
window.ToggleFullscreen();
```

#### 切换到边栏

**旧代码**:
```csharp
windowController.TogglePinnedDock();
```

**新代码**:
```csharp
window.ToggleSidebar();
```

#### 隐藏/显示窗口

**旧代码**:
```csharp
windowController.ToggleWindow();
```

**新代码**:
```csharp
window.ToggleWindowVisibility();
```

### 3. 监听状态变化（可选）

如果您需要在状态变化时执行自定义逻辑：

```csharp
// 在 MainWindow 构造函数中已经订阅了事件
// 您可以在事件处理器中添加自定义逻辑

private void OnWindowStateChanged(
    Docked_AI.功能.主窗口v2.状态机.WindowState from, 
    Docked_AI.功能.主窗口v2.状态机.WindowState to)
{
    // 添加您的自定义逻辑
    Console.WriteLine($"窗口状态从 {from} 变为 {to}");
}
```

## 📝 需要更新的文件

### 1. 更新 Linker 或 NavBar 中的按钮事件

找到全屏按钮和边栏按钮的点击事件，更新为调用新方法：

```csharp
// 在 Linker.xaml.cs 或 NavBar.xaml.cs 中
private void OnFullscreenButtonClick(object sender, RoutedEventArgs e)
{
    // 获取 MainWindow 实例
    var mainWindow = GetMainWindow();
    mainWindow.ToggleFullscreen(); // 使用新方法
}

private void OnSidebarButtonClick(object sender, RoutedEventArgs e)
{
    var mainWindow = GetMainWindow();
    mainWindow.ToggleSidebar(); // 使用新方法
}
```

### 2. 更新 TrayIconManager 中的托盘图标事件

```csharp
// 在 TrayIconManager.cs 中
private void OnTrayIconClick()
{
    if (_mainWindow != null)
    {
        _mainWindow.ToggleWindowVisibility(); // 使用新方法
    }
}
```

### 3. 更新快捷键处理器

```csharp
// 在快捷键处理器中
switch (hotkey.Key)
{
    case VirtualKey.F11:
        mainWindow.ToggleFullscreen(); // 使用新方法
        break;
    case VirtualKey.S when hotkey.Modifiers.HasFlag(VirtualKeyModifiers.Control | VirtualKeyModifiers.Shift):
        mainWindow.ToggleSidebar(); // 使用新方法
        break;
}
```

## ✅ 验证新架构是否工作

### 方法 1: 查看调试输出

运行应用后，在 Visual Studio 的输出窗口中查找以下日志：

```
MainWindow: WindowStateManager initialized successfully
MainWindow: Transitioned to Floating state after first frame
MainWindow: State changed from Initializing to Floating
```

如果看到这些日志，说明新架构正常工作！

### 方法 2: 测试状态切换

1. 启动应用
2. 点击全屏按钮（或按 F11）
3. 观察窗口是否平滑过渡到全屏
4. 再次点击，观察是否平滑返回浮窗模式

如果动画流畅，说明新架构正常工作！

## 🔧 故障排除

### 问题 1: 新架构未初始化

**症状**: 在输出窗口中看到 "Failed to initialize WindowStateManager"

**解决方案**:
1. 检查是否所有依赖项都已正确引用
2. 检查是否有编译错误
3. 查看详细的错误消息

**回退方案**: 新架构会自动回退到旧架构，应用仍然可以正常使用

### 问题 2: 按钮点击无响应

**症状**: 点击全屏或边栏按钮没有反应

**解决方案**:
1. 检查是否已更新按钮的点击事件处理器
2. 检查是否正确调用了 `ToggleFullscreen()` 或 `ToggleSidebar()`
3. 在事件处理器中添加断点，确认代码被执行

### 问题 3: 动画卡顿

**症状**: 窗口状态切换时动画不流畅

**解决方案**:
1. 检查系统性能（CPU、内存使用）
2. 查看动画性能日志（如果已实现）
3. 尝试调整动画时长（在 DefaultAnimationPolicy 中）

### 核心概念
- **声明式状态机**: 状态只定义"目标样子"，动画系统统一插值
- **请求压缩**: 多个快速连续的状态转换请求会被压缩为单个转换
- **平滑中断**: 打断时从当前视觉状态继续，无需反向动画

## 💡 提示

- **向后兼容**: 新架构包含了完整的向后兼容性，即使初始化失败也会回退到旧架构
- **线程安全**: 可以从任意线程调用 `TransitionTo`，内部会自动转发到 UI 线程
- **性能优化**: 动画使用时间驱动而非帧驱动，确保在不同性能的设备上都能准确完成
