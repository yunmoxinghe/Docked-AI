# 窗口大小和定位修复

## 问题描述

窗口大小不对，定位依据不对。

## 根本原因

之前的实现使用了固定的窗口尺寸（400x600）和错误的定位依据。

### 错误的实现

```csharp
const int defaultWidth = 400;
const int defaultHeight = 600;

var x = workArea.Right - defaultWidth - defaultMargin;
var y = workArea.Top + defaultMargin;
```

### 旧代码的正确逻辑

根据 `功能/主窗口/定位尺寸/窗口布局服务.cs`：

```csharp
// 宽度：工作区宽度的 1/3
int availableWidth = state.WorkArea.Right - state.WorkArea.Left - (state.Margin * 2);
state.WindowWidth = availableWidth / 3;
state.WindowWidth = Math.Max(state.MinWindowWidth, state.WindowWidth);
state.WindowWidth = Math.Min(availableWidth, state.WindowWidth);

// 高度：工作区高度减去上下边距
state.WindowHeight = state.WorkArea.Bottom - state.WorkArea.Top - (state.Margin * 2);

// 定位：基于工作区
state.TargetX = state.WorkArea.Right - state.WindowWidth - state.Margin;
state.TargetY = state.WorkArea.Top + state.Margin;
```

## 修复方案

### 正确的窗口尺寸计算

1. **宽度**：工作区宽度的 1/3
   - 最小宽度：380px
   - 最大宽度：工作区宽度减去左右边距

2. **高度**：工作区高度减去上下边距（各 10px）

3. **定位依据**：工作区（WorkArea），不是屏幕边界（MonitorBounds）

### 实现代码

```csharp
else
{
    // 首次启动，停靠在工作区右侧（与旧代码保持一致）
    var hwnd = _context.GetHwnd();
    var (monitorBounds, workArea) = WindowService.GetCurrentScreen(hwnd);
    
    const int defaultMargin = 10;
    const int minWindowWidth = 380;
    
    // 计算窗口尺寸（与旧代码逻辑一致）
    // 宽度：工作区宽度的 1/3
    int availableWidth = workArea.Width - (defaultMargin * 2);
    int windowWidth = availableWidth / 3;
    windowWidth = Math.Max(minWindowWidth, windowWidth);
    windowWidth = Math.Min(availableWidth, windowWidth);
    
    // 高度：工作区高度减去上下边距
    int windowHeight = workArea.Height - (defaultMargin * 2);
    
    // 定位：基于工作区（不是屏幕边界）
    // 停靠在工作区右侧，距离右边缘 10 像素
    var x = workArea.Right - windowWidth - defaultMargin;
    var y = workArea.Top + defaultMargin;
    
    return new WindowVisualState
    {
        Bounds = new Rect(x, y, windowWidth, windowHeight),
        CornerRadius = 12,
        Opacity = 1.0,
        IsTopmost = true,
        ExtendedStyle = 0
    };
}
```

## 关键区别

### 工作区 vs 屏幕边界

- **工作区（WorkArea）**：屏幕可用区域，不包括任务栏
- **屏幕边界（MonitorBounds）**：整个屏幕区域，包括任务栏

旧代码使用工作区作为定位依据，确保窗口不会被任务栏遮挡。

### 动态尺寸 vs 固定尺寸

- **旧代码**：根据工作区大小动态计算窗口尺寸
- **错误实现**：使用固定的 400x600 尺寸

动态尺寸确保窗口在不同分辨率的屏幕上都能正确显示。

## 示例计算

### 1920x1080 屏幕（任务栏高度 40px）

```
工作区：
- Left: 0
- Top: 0
- Right: 1920
- Bottom: 1040 (1080 - 40)
- Width: 1920
- Height: 1040

窗口尺寸：
- availableWidth = 1920 - 20 = 1900
- windowWidth = 1900 / 3 = 633
- windowWidth = max(380, 633) = 633
- windowHeight = 1040 - 20 = 1020

窗口位置：
- x = 1920 - 633 - 10 = 1277
- y = 0 + 10 = 10

最终窗口：(1277, 10, 633, 1020)
```

### 2560x1440 屏幕（任务栏高度 48px）

```
工作区：
- Left: 0
- Top: 0
- Right: 2560
- Bottom: 1392 (1440 - 48)
- Width: 2560
- Height: 1392

窗口尺寸：
- availableWidth = 2560 - 20 = 2540
- windowWidth = 2540 / 3 = 846
- windowWidth = max(380, 846) = 846
- windowHeight = 1392 - 20 = 1372

窗口位置：
- x = 2560 - 846 - 10 = 1704
- y = 0 + 10 = 10

最终窗口：(1704, 10, 846, 1372)
```
