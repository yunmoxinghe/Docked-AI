# 窗口样式说明

## WS_EX_TOOLWINDOW vs IsShownInSwitchers

### WS_EX_TOOLWINDOW（工具窗口样式）

**定义：** Win32 扩展窗口样式标志（0x00000080）

**效果：**
1. ✅ 窗口不显示在任务栏
2. ✅ 窗口不显示在 Alt+Tab 切换器中
3. ⚠️ 窗口标题栏变窄（如果有标题栏）
4. ⚠️ 窗口被系统视为"工具窗口"，可能影响某些行为
5. ⚠️ 可能影响窗口的 Z-order 行为

**适用场景：**
- 浮动工具面板
- 调色板窗口
- 辅助工具窗口
- 不需要在任务栏显示的小窗口

**缺点：**
- 语义上表示这是一个"工具窗口"，而不是主窗口
- 可能影响窗口的行为和外观
- 在某些情况下可能导致焦点管理问题

### IsShownInSwitchers（WinUI3 属性）

**定义：** WinUI3 AppWindow 的属性

**效果：**
1. ✅ 控制窗口是否显示在任务切换器（Alt+Tab）中
2. ✅ 不影响窗口的其他行为
3. ✅ 语义清晰，专门用于控制切换器显示
4. ❌ 不影响任务栏显示（窗口仍然可能显示在任务栏）

**适用场景：**
- 需要隐藏在任务切换器中，但保持正常窗口行为
- 主窗口但不希望在 Alt+Tab 中显示

**优点：**
- 语义清晰
- 不影响窗口的其他行为
- 更符合现代 Windows 应用的设计

## 当前实现的问题

### 问题 1：使用 WS_EX_TOOLWINDOW 不合适

当前 `FloatingWindow` 使用 `WS_EX_TOOLWINDOW`：

```csharp
ExtendedStyle = 窗口样式互操作.WS_EX_TOOLWINDOW
```

**问题：**
- Docked AI 的主窗口不是"工具窗口"，而是主应用窗口
- 使用工具窗口样式可能导致意外的行为
- 语义不正确

### 问题 2：与旧代码不一致

旧代码使用 `IsShownInSwitchers = false`：

```csharp
_window.AppWindow.IsShownInSwitchers = false;
```

新代码使用 `WS_EX_TOOLWINDOW`，导致行为不一致。

## 推荐方案

### 方案 1：移除 WS_EX_TOOLWINDOW，使用 IsShownInSwitchers

**优点：**
- 与旧代码一致
- 语义正确
- 不影响窗口的其他行为

**实现：**

1. 在 `FloatingWindow.GetTargetVisual()` 中移除 `ExtendedStyle`：
```csharp
return new WindowVisualState
{
    Bounds = targetBounds,
    CornerRadius = 12,
    Opacity = 1.0,
    IsTopmost = true,
    // ExtendedStyle = 窗口样式互操作.WS_EX_TOOLWINDOW  // 移除
};
```

2. 在 `FloatingWindow.OnEnter()` 中设置 `IsShownInSwitchers`：
```csharp
public void OnEnter()
{
    var window = _context.GetWindow();
    var hwnd = _context.GetHwnd();
    
    WindowService.ShowWindow(hwnd);
    WindowService.EnableResize(hwnd);
    WindowService.TryBringToFront(window);
    
    // 设置不在任务切换器中显示
    window.AppWindow.IsShownInSwitchers = false;
}
```

### 方案 2：保留 WS_EX_TOOLWINDOW，但明确用途

如果确实需要工具窗口样式（例如为了隐藏任务栏图标），应该：

1. 在代码中添加注释说明为什么使用工具窗口样式
2. 确保这是有意为之，而不是误用

**示例：**
```csharp
// 使用工具窗口样式以隐藏任务栏图标
// 注意：这会导致窗口被系统视为工具窗口，可能影响某些行为
ExtendedStyle = 窗口样式互操作.WS_EX_TOOLWINDOW
```

### 方案 3：根据窗口状态动态设置

不同的窗口状态可能需要不同的样式：

- **Floating**: 不使用工具窗口样式，使用 `IsShownInSwitchers = false`
- **Fullscreen**: 不使用工具窗口样式，可能需要显示在任务切换器中
- **Sidebar**: 可以使用工具窗口样式（因为边栏确实类似工具窗口）

## 任务栏显示控制

如果需要控制窗口是否显示在任务栏，有以下方法：

### 方法 1：WS_EX_TOOLWINDOW（当前使用）

```csharp
ExtendedStyle = 窗口样式互操作.WS_EX_TOOLWINDOW
```

**效果：** 窗口不显示在任务栏和任务切换器中

### 方法 2：WS_EX_APPWINDOW

```csharp
ExtendedStyle = 窗口样式互操作.WS_EX_APPWINDOW
```

**效果：** 强制窗口显示在任务栏（即使是工具窗口）

### 方法 3：组合使用

```csharp
// 显示在任务栏，但不显示在任务切换器
ExtendedStyle = 窗口样式互操作.WS_EX_APPWINDOW
window.AppWindow.IsShownInSwitchers = false;
```

## 建议

对于 Docked AI 主窗口：

1. **移除 WS_EX_TOOLWINDOW** - 这不是工具窗口
2. **使用 IsShownInSwitchers = false** - 控制任务切换器显示
3. **如果需要隐藏任务栏图标** - 考虑使用托盘图标而不是工具窗口样式
4. **保持与旧代码一致** - 避免引入意外的行为变化
