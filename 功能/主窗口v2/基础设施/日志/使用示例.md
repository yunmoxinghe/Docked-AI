# 日志系统使用示例

本文档展示如何在主窗口重构项目中使用日志系统。

## 概述

日志系统提供了三个专门的日志记录器：

1. **StateTransitionLogger** - 记录窗口状态转换的完整生命周期
2. **AnimationPerformanceLogger** - 记录动画执行的性能指标
3. **ResourceManagementLogger** - 记录资源（如 CancellationTokenSource）的创建和释放

## 基本用法

### 1. 创建日志记录器

```csharp
using Docked_AI.功能.主窗口v2.基础设施.日志;

// 创建基础日志记录器
var logger = new DebugLogger("WindowManager");

// 创建专门的日志记录器
var stateLogger = new StateTransitionLogger(logger);
var perfLogger = new AnimationPerformanceLogger(logger);
var resourceLogger = new ResourceManagementLogger(logger);
```

### 2. 在 WindowStateManager 中使用

```csharp
// 创建 WindowStateManager 时传入日志记录器
var stateManager = new WindowStateManager(
    dispatcher: DispatcherQueue.GetForCurrentThread(),
    animationEngine: animationEngine,
    context: windowContext,
    animationPolicy: animationPolicy,
    logger: logger  // 传入日志记录器
);
```

WindowStateManager 会自动记录：
- 状态转换开始（TransitionStarted）
- 状态转换完成（StateChanged）
- 状态转换失败（TransitionFailed）
- 状态转换被取消（Cancelled）
- 重试尝试
- 关键故障
- CancellationTokenSource 的创建、取消和释放

### 3. 在 AnimationEngine 中使用

```csharp
// 创建 AnimationEngine 时传入日志记录器
var animationEngine = new AnimationEngine(logger);
```

AnimationEngine 会自动记录：
- 动画开始时间和类型
- 动画完成时的实际持续时间、帧数和帧率
- 动画被中断时的已执行时间和帧数
- 动画时长偏差（如果超过 50ms）

## 性能监控

AnimationPerformanceLogger 会自动记录以下性能指标：

1. **动画持续时间** - 实际执行时间 vs 计划时间
2. **帧率 (FPS)** - 动画期间的平均帧率
3. **帧数** - 动画执行的总帧数
4. **时长偏差** - 如果实际时长与计划时长偏差超过 50ms，会记录警告

这些指标可以帮助：
- 识别性能瓶颈
- 检测帧率下降
- 验证动画时长准确性
- 优化动画参数

## 自定义日志记录器

如果需要将日志输出到文件或其他目标，可以实现 `ILogger` 接口：

```csharp
public class FileLogger : ILogger
{
    private readonly string _logFilePath;
    
    public FileLogger(string logFilePath)
    {
        _logFilePath = logFilePath;
    }
    
    public void Debug(string message)
    {
        WriteToFile($"[DEBUG] {message}");
    }
    
    public void Info(string message)
    {
        WriteToFile($"[INFO] {message}");
    }
    
    public void Warning(string message)
    {
        WriteToFile($"[WARNING] {message}");
    }
    
    public void Error(string message, Exception? exception = null)
    {
        WriteToFile($"[ERROR] {message}");
        if (exception != null)
        {
            WriteToFile($"[ERROR] Exception: {exception}");
        }
    }
    
    public void Performance(string metric, double value, string? unit = null)
    {
        var unitStr = unit != null ? $" {unit}" : "";
        WriteToFile($"[PERF] {metric}: {value:F2}{unitStr}");
    }
    
    private void WriteToFile(string message)
    {
        File.AppendAllText(_logFilePath, $"{DateTime.Now:yyyy-MM-dd HH:mm:ss.fff} {message}\n");
    }
}
```

然后使用自定义日志记录器：

```csharp
var logger = new FileLogger("window_manager.log");
var stateManager = new WindowStateManager(
    dispatcher: DispatcherQueue.GetForCurrentThread(),
    animationEngine: new AnimationEngine(logger),
    context: windowContext,
    animationPolicy: animationPolicy,
    logger: logger
);
```

## 禁用日志

如果不需要日志记录，只需在创建 WindowStateManager 和 AnimationEngine 时不传入 logger 参数：

```csharp
// 不传入 logger 参数，日志功能将被禁用
var stateManager = new WindowStateManager(
    dispatcher: DispatcherQueue.GetForCurrentThread(),
    animationEngine: new AnimationEngine(),  // 无日志
    context: windowContext,
    animationPolicy: animationPolicy
    // 不传入 logger 参数
);
```

## 最佳实践

1. **开发阶段**：使用 `DebugLogger` 输出到调试控制台
2. **生产环境**：使用 `FileLogger` 或其他持久化日志记录器
3. **性能测试**：启用 `AnimationPerformanceLogger` 收集性能指标
4. **故障排查**：启用所有日志记录器以获取完整的诊断信息
5. **发布版本**：考虑禁用或减少日志级别以提高性能

## 注意事项

- 日志记录是可选的，不传入 logger 参数时不会影响功能
- 日志记录器是线程安全的（通过 DispatcherQueue 确保）
- 性能日志不会显著影响动画性能（开销 < 1ms）
- 建议在开发和测试阶段启用所有日志，在生产环境中根据需要调整
