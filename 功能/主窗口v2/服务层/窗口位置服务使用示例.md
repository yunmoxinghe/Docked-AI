# WindowPositionService 使用示例

## 概述

`WindowPositionService` 实现了 `IWindowPositionService` 接口，使用 Windows.Storage.ApplicationData.Current.LocalSettings 持久化存储浮窗位置、尺寸和边缘距离。

## 创建实例

```csharp
// 创建窗口位置服务实例
var positionService = new WindowPositionService();
```

## 保存浮窗位置

```csharp
// 保存当前浮窗的位置和尺寸
// 参数：宽度、高度、距屏幕右边缘的距离、距屏幕下边缘的距离
positionService.SaveFloatingPosition(
    width: 400,
    height: 600,
    rightDistance: 10,
    bottomDistance: 10
);
```

## 恢复浮窗位置

```csharp
// 获取上次保存的浮窗位置
var lastPosition = positionService.GetLastFloatingPosition();

if (lastPosition != null)
{
    // 使用保存的位置信息
    var width = lastPosition.Width;
    var height = lastPosition.Height;
    var rightDistance = lastPosition.RightDistance;
    var bottomDistance = lastPosition.BottomDistance;
    
    // 根据保存的边缘距离和当前屏幕尺寸计算窗口位置
    var x = screenWidth - rightDistance - width;
    var y = screenHeight - bottomDistance - height;
}
else
{
    // 首次启动，使用默认值
    var width = 400;
    var height = 600;
    // 居中显示
    var x = (screenWidth - width) / 2;
    var y = (screenHeight - height) / 2;
}
```

## 在 FloatingWindow 中的使用

`FloatingWindow` 类已经集成了 `IWindowPositionService`：

```csharp
public class FloatingWindow : IWindowState
{
    private readonly IWindowPositionService _positionService;

    public FloatingWindow(WindowContext context, IWindowPositionService positionService)
    {
        _positionService = positionService;
    }

    public WindowVisualState GetTargetVisual()
    {
        // 恢复上次位置
        var lastPosition = _positionService.GetLastFloatingPosition();
        if (lastPosition != null)
        {
            // 使用保存的位置
        }
        else
        {
            // 使用默认位置
        }
    }

    public void OnExit()
    {
        // 保存当前位置
        _positionService.SaveFloatingPosition(
            currentVisual.Bounds.Width,
            currentVisual.Bounds.Height,
            rightDistance,
            bottomDistance
        );
    }
}
```

## 依赖注入示例

在应用启动时创建服务实例：

```csharp
// 在 App.xaml.cs 或 MainWindow 构造函数中
var positionService = new WindowPositionService();

// 创建 FloatingWindow 时注入服务
var floatingWindow = new FloatingWindow(windowContext, positionService);
```

## 线程安全

`WindowPositionService` 使用 `lock` 确保线程安全，可以从任意线程调用：

```csharp
// 从 UI 线程调用
positionService.SaveFloatingPosition(400, 600, 10, 10);

// 从后台线程调用（也是安全的）
Task.Run(() => {
    var lastPosition = positionService.GetLastFloatingPosition();
});
```

## 异常处理

服务内部捕获所有异常，不会抛出异常影响应用运行：

```csharp
// 如果保存失败，会记录调试信息但不抛出异常
positionService.SaveFloatingPosition(400, 600, 10, 10);

// 如果读取失败，返回 null
var lastPosition = positionService.GetLastFloatingPosition();
if (lastPosition == null)
{
    // 使用默认值
}
```

## 存储位置

数据存储在 Windows 应用程序本地设置中：

- 路径：`%LocalAppData%\Packages\<PackageFamilyName>\Settings\settings.dat`
- 键名：
  - `FloatingWindow_Width`: 窗口宽度
  - `FloatingWindow_Height`: 窗口高度
  - `FloatingWindow_RightDistance`: 距屏幕右边缘的距离
  - `FloatingWindow_BottomDistance`: 距屏幕下边缘的距离

## 数据验证

服务会验证读取的数据：

- 宽度和高度必须大于 0
- 右边距和下边距必须大于等于 0
- 所有值必须是 `double` 类型
- 如果验证失败，返回 `null`

## 测试建议

### 手动测试步骤

1. **首次启动测试**：
   - 启动应用，窗口应显示在默认位置（居中，400x600）
   - 调整窗口大小和位置
   - 关闭应用

2. **位置恢复测试**：
   - 重新启动应用
   - 验证窗口恢复到上次的位置和尺寸

3. **多显示器测试**：
   - 在多显示器环境中移动窗口
   - 关闭应用
   - 重新启动应用
   - 验证窗口位置正确恢复

4. **分辨率变化测试**：
   - 保存窗口位置
   - 更改屏幕分辨率
   - 重新启动应用
   - 验证窗口仍然可见（至少部分可见）

### 单元测试示例（伪代码）

```csharp
[Test]
public void SaveAndRestore_ShouldPersistPosition()
{
    var service = new WindowPositionService();
    
    // 保存位置
    service.SaveFloatingPosition(400, 600, 10, 10);
    
    // 恢复位置
    var position = service.GetLastFloatingPosition();
    
    Assert.NotNull(position);
    Assert.Equal(400, position.Width);
    Assert.Equal(600, position.Height);
    Assert.Equal(10, position.RightDistance);
    Assert.Equal(10, position.BottomDistance);
}

[Test]
public void GetLastFloatingPosition_WhenNoData_ShouldReturnNull()
{
    var service = new WindowPositionService();
    
    // 清除所有数据
    ApplicationData.Current.LocalSettings.Values.Clear();
    
    // 获取位置
    var position = service.GetLastFloatingPosition();
    
    Assert.Null(position);
}
```

## 注意事项

1. **首次启动**：首次启动时 `GetLastFloatingPosition()` 返回 `null`，应使用默认值
2. **屏幕边界**：调用方需要确保计算的窗口位置在屏幕范围内
3. **线程安全**：服务内部已处理线程安全，无需外部加锁
4. **异常处理**：服务内部捕获所有异常，不会影响应用运行
5. **数据持久化**：数据自动持久化到 Windows 应用程序本地设置，无需手动保存
