#### 幂等性模式

确保每个 OnEnter 实现遵循以下模式：

```csharp
public void OnEnter()
{
    // 1. 使用标志防止重复操作
    if (_isEntered)
    {
        return;
    }
    _isEntered = true;
    
    // 2. 或者使用检查-设置模式
    var window = _context.GetWindow();
    if (!window.IsVisible)
    {
        window.IsVisible = true;
    }
    
    // 3. 对于资源注册，先清理再注册
    UnregisterAppBar(); // 清理可能存在的旧注册
    RegisterAppBar();   // 注册新的
}
```

#### 需要修改的示例

**边栏窗口 (SidebarWindow)**:

```csharp
private bool _isAppBarRegistered = false;

public void OnEnter()
{
    var window = _context.GetWindow();
    var hwnd = _context.GetHwnd();
    
    // 幂等设置
    if (!window.IsVisible)
    {
        window.IsVisible = true;
    }
    
    if (!window.IsHitTestVisible)
    {
        window.IsHitTestVisible = true;
    }
    
    // AppBar 注册（防止重复注册）
    if (!_isAppBarRegistered)
    {
        WindowService.RegisterAppBar(hwnd, AppBarEdge.Right, 400);
        _isAppBarRegistered = true;
    }
    
    WindowService.DisableResize(hwnd);
    WindowService.TryBringToFront(window);
}

public void OnExit()
{
    if (_isAppBarRegistered)
    {
        var hwnd = _context.GetHwnd();
        WindowService.UnregisterAppBar(hwnd);
        _isAppBarRegistered = false;
    }
}
```

### 任务 14.2: 记录中断时不调用 OnExit

**需求**: 添加代码注释说明动画取消时不调用 OnExit

#### 在 WindowStateManager.cs 中添加注释

在 `RunStateMachineLoop` 方法的 catch 块中添加注释：

```csharp
catch (OperationCanceledException)
{
    // ========================================
    // 重要：动画被打断时不调用 OnExit
    // ========================================
    // 原因：
    // 1. 状态尚未稳定，不应该执行清理操作
    // 2. 下一轮循环会调用新目标状态的 OnEnter
    // 3. OnEnter 必须能够处理先前 OnEnter 资源的清理（幂等性）
    //
    // 示例场景：
    // - 用户快速切换 Floating → Sidebar → Fullscreen
    // - Floating 的 OnEnter 被调用
    // - 动画被打断，Floating 的 OnExit 不会被调用
    // - Sidebar 的 OnEnter 被调用（必须能够清理 Floating 的资源）
    // - 动画再次被打断，Sidebar 的 OnExit 不会被调用
    // - Fullscreen 的 OnEnter 被调用（必须能够清理 Sidebar 的资源）
    
    TransitioningTo = null;
    continue;
}
```

using System.Diagnostics;

namespace Docked_AI.功能.主窗口v2.基础设施.日志
{
    public class AnimationPerformanceLogger
    {
        private readonly Stopwatch _stopwatch = new Stopwatch();
        private int _frameCount = 0;
        private DateTime _startTime;
        
        public void Start()
        {
            _stopwatch.Restart();
            _frameCount = 0;
            _startTime = DateTime.Now;
        }
        
        public void RecordFrame()
        {
            _frameCount++;
        }
        
        public void Stop(bool wasCancelled)
        {
            _stopwatch.Stop();
            var duration = _stopwatch.Elapsed;
            var fps = _frameCount / duration.TotalSeconds;
            
            Debug.WriteLine($"[AnimationPerf] Duration: {duration.TotalMilliseconds:F2}ms");
            Debug.WriteLine($"[AnimationPerf] Frames: {_frameCount}");
            Debug.WriteLine($"[AnimationPerf] FPS: {fps:F2}");
            Debug.WriteLine($"[AnimationPerf] Cancelled: {wasCancelled}");
        }
    }
}

using Microsoft.VisualStudio.TestTools.UnitTesting;
using System.Threading.Tasks;

namespace Docked_AI.Tests
{
    [TestClass]
    public class WindowStateManagerTests
    {
        [TestMethod]
        public async Task TestRapidStateTransitions()
        {
            // 安排
            var window = CreateTestWindow();
            var stateManager = CreateStateManager(window);
            
            var stateChanges = new List<(WindowState from, WindowState to)>();
            stateManager.StateChanged += (from, to) => stateChanges.Add((from, to));
            
            // 执行：快速触发多个状态转换
            stateManager.TransitionTo(WindowState.Floating);
            stateManager.TransitionTo(WindowState.Fullscreen);
            stateManager.TransitionTo(WindowState.Sidebar);
            stateManager.TransitionTo(WindowState.Floating);
            
            // 等待所有转换完成
            await Task.Delay(2000);
            
            // 断言
            Assert.AreEqual(WindowState.Floating, stateManager.CurrentState);
            Assert.IsNull(stateManager.TransitioningTo);
            
            // 验证只有最终状态变化被记录
            Assert.AreEqual(1, stateChanges.Count);
            Assert.AreEqual(WindowState.Initializing, stateChanges[0].from);
            Assert.AreEqual(WindowState.Floating, stateChanges[0].to);
        }
    }
}

## 核心概念

### 声明式状态机

不同于传统的命令式状态机，v2 采用声明式设计：
- 状态只定义"目标样子"
- 动画系统统一插值
- 打断时无需反向动画

### 请求压缩

多个快速连续的状态转换请求会被压缩为单个转换：
- 只保留最后一个目标状态
- 中间状态不会成为稳定状态
- 动画从当前视觉状态平滑过渡到最终目标
